#### 1. `board_api.md` – Board API 接口说明

```markdown
# Board API 接口说明

---

## 1. 概览

Board API 由两个核心抽象类组成：

- `Position`：表示题板上的单个格子位置，包含坐标及题板标识，提供位置移动、偏移、邻域查询和边界判断。
- `Board`：表示整个题板，管理多个子题板（`board_key`）的内容、变量、配置及渲染功能。

该接口设计用于支持多题板、多位置、多变量的复杂谜题状态管理，确保位置和内容操作的统一和高效。

---

## 2. 结构划分

| 小节         | 内容说明                          |
| ------------ | -------------------------------- |
| `Position`   | 单个位置抽象，坐标及操作接口        |
| `Board`      | 题板整体抽象，数据管理、变量接口、配置管理及特殊遍历等功能 |

---

## 3. `Position` 接口

### 属性

| 名称         | 类型   | 说明                    |
| ------------ | ------ | ----------------------- |
| `row`        | `int`  | 行索引（纵向坐标）       |
| `col`        | `int`  | 列索引（横向坐标）       |
| `board_key`  | `str`  | 所属题板标识             |

> **兼容性**：`x` 和 `y` 属性已弃用，请使用 `row` 和 `col`。

### 方法

| 方法                                     | 返回类型       | 说明                                                         |
| ---------------------------------------- | -------------- | ------------------------------------------------------------ |
| `clone()`                                | `Position`     | 复制当前对象，返回同坐标同题板的新位置。                       |
| `up(n=1)`, `down(n=1)`, `left(n=1)`, `right(n=1)` | `Position`     | 返回向对应方向移动 `n` 格的新位置，不修改自身。               |
| `shift(col=0, row=0)`                    | `Position`     | 返回按指定行列偏移后的新位置。                                |
| `deviation(pos)`                         | `Position`     | 返回以 `pos` 为偏移量的新位置。                              |
| `neighbors(*args)`                       | `list[Position]` | 返回距离满足条件的邻格列表。支持一个参数（最大距离平方）或两个参数（最小、最大距离平方）。 |
| `in_bounds(bound_pos)`                   | `bool`         | 判断自身是否在 `bound_pos` 定义的边界内（含边界）。           |

> **距离说明**：使用欧几里得距离的平方值判断，如 `neighbors(2)` 返回距离 ≤ √2 的格子（即上下左右及对角）。

---

## 4. `Board` 接口

### 4.1 类属性

| 名称            | 类型         | 说明                                                         |
| --------------- | ------------ | ------------------------------------------------------------ |
| `CONFIG_FLAGS`  | `list[str]`  | 预定义配置项名称：`"by_mini"`（角标）、`"pos_label"`（X=N 标签）、`"row_col"`（行列号）、`"interactive"`（是否参与求解交互）。 |

### 4.2 核心数据访问

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| `get_value(pos)`, `set_value(pos, value)`    | 获取/设置指定位置的对象（线索或雷）。                        |
| `get_type(pos, special='')`                  | 返回类型字符串（`'C'`/`'F'`/`'N'`）。支持 `special` 参数用于自定义类型映射（需提前注册）。 |
| `get_dyed(pos)`, `set_dyed(pos, dyed)`       | 获取/设置染色状态（布尔）。                                  |
| `get_variable(pos, special='')`              | 获取对应位置的 CP-SAT 变量（`IntVar`），支持 `special` 参数获取自定义变量。 |
| `batch(positions, mode, drop_none=False)`    | 批量获取多个位置的信息，`mode` 可选 `'obj'`, `'type'`, `'var'`, `'dye'`。 |

### 4.3 遍历与迭代

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| `__call__(target='always', mode='obj', key=None)` | 生成器，遍历符合条件的格子。`target` 可设为 `'C'`、`'F'`、`'N'` 或 `'always'`；`mode` 决定返回内容（对象/类型/变量/染色/`None`）。 |
| `has(target, key=None)`                     | 判断指定题板（或全部）中是否存在 `target` 类型的元素。        |

### 4.4 题板管理

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| `generate_board(board_key, size, labels=None, true_tag=None, false_tag=None)` | 创建新的子题板，可指定默认雷/线索值对象。                     |
| `get_board_keys()`, `get_interactive_keys()` | 返回所有题板键 / 仅 `interactive=True` 的题板键。           |
| `get_config(board_key, config_name)`, `set_config(...)` | 读写配置（尺寸、标签、标志等）。                             |
| `boundary(key=MASTER_BOARD_KEY) -> Position` | 返回题板的右下角边界位置（含边界格）。                       |
| `is_valid(pos)`, `in_bounds(pos)`            | 判断位置是否在题板内且未被掩码（`mask`）排除。               |

### 4.5 特殊类型注册

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| `register_type_special(name, func)`         | 注册一个自定义类型获取函数，用于 `get_type` 的 `special` 参数。函数签名：`(board, pos, *args, **kwargs) -> str`。 |
| `has_type_special(name) -> bool`            | 检查是否已注册某个 `special` 名称。                          |

### 4.6 序列化与克隆

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| `clone() -> Board`                          | 深拷贝当前题板（通过 JSON 序列化实现）。                     |
| `json() -> JSONObject`                      | 将整个题板状态序列化为 JSON 对象（含所有子题板、单元格、配置）。 |
| `from_json(data) -> Board`                  | 从 JSON 对象反序列化恢复题板。                              |
| `serialize() -> str`                        | 返回压缩后的 JSON 字符串（便于存储/传输）。                  |

---

## 5. 扩展点

- **自定义位置类**：继承 `Position` 可重写移动或邻域逻辑。
- **自定义题板行为**：继承 `Board` 可重写 `get_type`、`get_variable` 或添加新配置项。
- **注册特殊类型**：使用 `register_type_special` 可让规则在 `get_type` 中动态计算类型，而无需存储实际对象。

---
```

# 右线规则接口说明

---

## 1. 概述

右线规则（Clue）负责表示谜题中数字线索的行为和约束。每个规则由两部分组成：**规则类**（定义线索的整体逻辑与填充）和**线索值类**（定义单个线索的约束、渲染和推理）。

本文档适用于希望扩展或实现自定义右线规则的开发者。

---

## 2. 类结构概览

| 类名                  | 继承自                           | 职责                         |
| --------------------- | -------------------------------- | ---------------------------- |
| `AbstractClueRule`    | `abs.Rrule.AbstractClueRule`     | 规则整体，负责填充线索与整体建模 |
| `AbstractClueValue`   | `abs.Rrule.AbstractClueValue`    | 单个线索格的具体数值行为       |

---

## 3. `AbstractClueRule` 接口

### 3.1 类属性（元信息）

所有属性均为类级别定义，用于文档和自动发现。

| 属性            | 类型                  | 说明                                                         |
| --------------- | --------------------- | ------------------------------------------------------------ |
| `id`            | `str`                 | 规则唯一标识（如 `"V"`, `"1E"`）。                           |
| `name`          | `I18n` 或 `str`       | 规则名称，支持国际化（如 `name.zh_CN = "标准扫雷"`）。       |
| `doc`           | `I18n` 或 `str`       | 规则详细描述，支持多语言。                                   |
| `author`        | `tuple[str, int]`     | 作者名和 ID。                                                |
| `tags`          | `list[Tag]`           | 预定义标签（如 `"Original"`, `"Local"`, `"Number Clue"`）。 |
| `creation_time` | `str`                 | 规则创建时间（格式不限）。                                   |
| `lib_only`      | `bool`                | 若为 `True`，规则不会单独出现在用户规则列表中，仅作为依赖。  |

> **国际化支持**：`name` 和 `doc` 可赋予 `I18n` 对象，支持通过 `属性.语言代码` 设置（如 `name.zh_CN = "视野"`）。系统会根据当前 locale 自动选择合适翻译。

### 3.2 核心方法

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| `fill(board: Board) -> Board`               | **必须实现**。遍历 `board("N")`（未定义格），根据当前雷区（`board` 中已有雷）为每个格子设置线索值对象（继承自 `AbstractClueValue`）。 |
| `create_constraints(board: Board, switch: Switch)` | 为规则整体添加约束（如全局计数、动态删线索状态管理等）。通常无需重写，因为约束可以放在线索值对象中。 |
| `init_clear(board: Board)`                  | 在生成阶段调用，可用于清除由该规则产生的特殊标记（如副板内容）。 |
| `init_board(board: Board)`                  | 在生成完整答案板后调用，用于补充副板内容（如生成 `2E` 的加密副板）。 |
| `suggest_total(info: SuggestionInfo)`       | 建议总雷数或添加硬/软约束。详见 [utils.md](#suggest_total)。 |
| `combine(rules)`                            | 在规则加载后调用，允许规则感知其他规则并进行联合优化。        |
| `get_deps() -> list[str]`                   | 返回依赖的其他规则 ID 列表，系统会自动按顺序加载。           |

### 3.3 动态删线索支持（可选）

若规则支持“动态删线索”模式（即生成谜题时逐步删除线索并保持唯一解），需设置以下属性和方法：

| 属性/方法                                    | 说明                                                         |
| -------------------------------------------- | ------------------------------------------------------------ |
| `dynamic_dig_enabled = True`                | 启用动态删线索流程。                                         |
| `dynamic_dig_use_visibility_optimizer = True` | （可选）启用基于求解器的优化路径，可减少搜索时间。            |
| `dynamic_init_visibility(board, visibility_state)` | 初始化可见性状态（`visibility_state` 为 `dict[str, dict[tuple[int,int], bool | None]]`，表示每个子题板每个格子的显隐状态）。 |
| `dynamic_on_visibility_changed(board, visibility_state, changed_positions)` | 当某些格子的显隐状态改变时调用，用于更新约束或缓存。          |

---

## 4. `AbstractClueValue` 接口

### 4.1 类属性

| 属性   | 类型   | 说明                         |
| ------ | ------ | ---------------------------- |
| `id`   | `str`  | 线索值类型标识，通常与规则 `id` 一致。 |

### 4.2 必须实现的方法

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| `create_constraints(board: Board, switch: Switch)` | **核心**。为该线索添加 CP-SAT 约束，例如“周围八格雷数等于某值”。通过 `switch.get(model, self)` 获取激活变量。 |
| `type() -> bytes`                           | 返回线索类型字节码（通常为 `self.id.encode('ascii')`）。     |
| `code() -> bytes`                           | 返回该线索实例的序列化编码（用于保存/加载）。                |

### 4.3 可选实现的方法

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| `__repr__() -> str`                         | 返回简短字符串表示，用于调试和文本输出。                     |
| `compose(board) -> Element`                 | 返回图像渲染元素（如 `get_text` 或 `get_image`）。默认使用 `__repr__`。 |
| `web_component(board) -> Element`           | 返回 Web 渲染元素（与 `compose` 类似，但可针对 Web 优化）。 |
| `high_light(board) -> list[Position]`       | 返回该线索影响的高亮区域（用于图像提示）。                   |
| `invalid(board) -> bool`                    | 判断该线索是否已失效（如周围无未知格），用于优化约束。       |
| `deduce_cells(board) -> bool`               | 尝试进行局部推理并修改题板（如标注必雷/必空），返回是否修改。 |
| `tag(board) -> str`                         | 返回用于角标的标签字符串（默认为 `self.id`）。               |
| `weaker(board) -> AbstractValue`            | 返回一个更弱的线索值对象（用于动态删线索时逐渐弱化线索）。   |
| `weaker_times() -> int`                     | 返回可被弱化的次数（`weaker` 可调用次数）。                  |

## 5. 实现位置与命名规范

- 规则类文件放置于 `impl/rule/Rrule/` 目录下。
- 文件名应与规则 `id` 一致（如 `V.py`、`1E.py`）。
- 规则类建议命名为 `RuleXXX`，线索值类命名为 `ValueXXX`。
- 文件顶部 **必须包含模块级 docstring**，作为规则的简短说明（用于 `run list` 展示）。

示例：

```python
"""
[V] 标准扫雷：每个数字标明周围八格内雷的数量。
"""
from minesweepervariants.abs.Rrule import AbstractClueRule, AbstractClueValue

class RuleV(AbstractClueRule):
    id = "V"
    name = "Vanilla"
    doc = "Each number indicates the number of mines in the surrounding eight cells"
    # ... 
```

---

## 6. 使用 `Switch` 类

在 `create_constraints` 中，通过 `switch.get(model, obj, index=None)` 获取一个布尔变量，用于控制该约束是否生效。常见用法：

```python
def create_constraints(self, board, switch):
    model = board.get_model()
    s = switch.get(model, self)          # 为当前线索值对象获取开关
    model.Add(sum(neighbor_vars) == self.value).OnlyEnforceIf(s)
```

若同一规则需要多个子开关（如区分行列），可传入 `index` 参数：

```python
s_row = switch.get(model, self, "row")
s_col = switch.get(model, self, "col")
```

## 7. 序列化与反序列化

`AbstractClueValue` 的子类应实现 `json()` 和 `from_json` 方法（默认会使用 `code()` 和 `type()` 进行编码）。若需存储额外信息，可覆盖：

```python
def json(self):
    return {"value": self.count, "extra": self.extra}

@classmethod
def from_json(cls, pos, data):
    return cls(pos, data["value"], data["extra"])
```

## 1. 概述

# 左线规则接口说明

---

## 1. 概述

左线规则（Mines）定义地雷的全局布局约束，是题板生成的核心。每个左线规则通过 `create_constraints` 方法向 CP-SAT 模型添加约束，确保生成的雷区满足特定拓扑或计数条件。

---

## 2. 类结构

| 类名                  | 继承自                           | 职责                         |
| --------------------- | -------------------------------- | ---------------------------- |
| `AbstractMinesRule`   | `abs.Lrule.AbstractMinesRule`    | 单个地雷布局规则              |
| `MinesRules`          | （容器）                         | 管理多个左线规则的组合         |

---

## 3. `AbstractMinesRule` 接口

### 类属性（元信息）

与右线规则相同：`id`, `name`, `doc`, `author`, `tags`, `creation_time`, `lib_only`。支持国际化（`I18n`）。

### 核心方法

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| `create_constraints(board: Board, switch: Switch)` | **必须实现**。向 CP-SAT 模型添加约束。通过 `switch.get(model, self)` 获取该规则的激活变量（通常用于规则组合或元规则）。 |
| `suggest_total(info: SuggestionInfo)`       | 建议总雷数或添加硬/软约束。                                  |
| `init_board(board: Board)`                  | 生成答案板后调用，可用于初始化副板内容。                     |
| `init_clear(board: Board)`                  | 生成谜题前调用，用于清理副板或标记。                         |
| `combine(rules)`                            | 规则组合优化，可感知其他规则并调整约束（如 `2B` 会检测是否与 `1B` 同时存在）。 |
| `get_deps() -> list[str]`                   | 依赖规则列表。                                               |

---

## 4. 约束建模要点

- 使用 `board.get_variable(pos, special='raw')` 获取雷变量（0/1）。
- 通过 `board.get_model()` 获取 `CpModel` 实例。
- 使用 `model.Add(...)` 添加约束，并通过 `.OnlyEnforceIf(switch_var)` 使约束受开关控制。
- 可创建辅助变量（`model.NewBoolVar` 或 `NewIntVar`）表达复杂逻辑。

### 示例：`1A`（无马步）

```python
def create_constraints(self, board, switch):
    model = board.get_model()
    s = switch.get(model, self)
    for pos, var in board(mode="variable"):
        for neighbor in [pos.left(2).up(1), ...]:   # 马步位置
            nvar = board.get_variable(neighbor)
            if nvar is not None:
                model.AddBoolOr([var.Not(), nvar.Not()]).OnlyEnforceIf(s)
```

## 5. `MinesRules` 容器

用于存储多个左线规则，并提供统一接口：

- `append(rule)` – 添加规则
- `rules` – 列表属性

通常由 `Summon` 内部管理，无需直接使用。

## 6. 扩展建议

- 优先使用 CP-SAT 的 `AddBoolOr`, `AddBoolAnd`, `AddSumEquality` 等原生约束。

- 若规则需要大量枚举候选，请考虑使用 `model.AddAllowedAssignments` 或预计算布尔变量。

## 3. 核心接口

### `Switch` 类（位于 `impl.summon.solver`）

**职责**：管理规则和线索的激活布尔变量，支持对同一对象获取多个不同索引的开关。

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| `get(model, obj, index=None) -> IntVar`     | 获取或创建一个与 `obj`（规则、位置或值对象）关联的布尔变量。 |
| `get_all_vars() -> list[IntVar]`            | 返回所有已创建的变量。                                       |
| `get_switches_by_obj(obj)`                  | 返回某个对象的所有开关列表，格式为 `[(index, var), ...]`。   |
| `get_obj_and_index_by_var(var)`             | 通过变量反查其所属的对象字符串和索引。                       |

> **用法**：在 `create_constraints` 中通过 `switch.get(model, self)` 获取当前约束的激活变量，然后使用 `.OnlyEnforceIf(switch_var)` 将约束置于该开关之下。

## 开发指南

### 新增规则

1. 确定规则类型（左线/右线/中线），在 `impl/rule/{Lrule|Rrule|Mrule}/` 下创建新的 Python 文件。
2. 继承对应的抽象类，实现必须方法。
3. 设置类属性（`id`, `name`, `doc` 等），并确保文件顶部有模块 docstring。
4. （可选）实现 `combine`、`suggest_total` 等优化方法。
5. 运行 `poetry run python -m minesweepervariants list` 验证规则是否被正确识别。

### 新增染色或掩码规则

- 在 `impl/board/dye/` 下创建文件，继承 `AbstractDye`，实现 `dye(board)` 方法。
- 通过 `board.set_dyed(pos, bool)` 设置染色状态。
- 掩码规则（`mask`）逻辑类似，通过 `board.set_mask(pos)` 标记无效格。

### 调试与测试

- 使用 `-L TRACE` 启用详细日志，观察约束添加与求解过程。
- 利用 `Summon` 和 `GameSession` 的交互模式验证规则正确性。
